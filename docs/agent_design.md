# 南海区数据产业企业图谱 Agent 流程设计

## 1. 文档目的

本文件用于定义“南海区数据产业企业图谱”项目中的智能体（Agent）流程，明确企业发现、信息抽取、分类、校验和人工复核的任务分工，为后续自动化扩展与系统实现提供依据。

---

## 2. 总体流程概述

本项目采用“多智能体协同 + 人工复核”的流程，对南海区数据产业企业进行发现、抽取、分类和校验。

整体流程如下：

候选企业发现
→ 企业信息抽取
→ 企业分类
→ 数据校验
→ 人工复核
→ 写入数据库 / 前端展示

---

## 3. Agent 设计

### Agent1：企业发现

**输入：**

- 关键词
- 地区（如：南海区、桂城街道、狮山镇等）

**功能：**

- 根据地区和关键词搜索候选企业
- 生成企业候选名单
- 收集初步来源链接

**输出：**

- 企业列表
- 初始来源链接
- 企业名称关键词集合

**示例：**
输入：

- 关键词：大数据、数据服务、工业互联网
- 地区：南海区

输出：

- 候选企业A
- 候选企业B
- 候选企业C

---

### Agent2：信息抽取

**输入：**

- 网页内容
- 企业官网介绍
- 新闻报道
- 招聘信息
- 行业文章

**功能：**

- 从网页文本中抽取企业核心字段
- 统一转化成结构化数据

**输出：**

- 企业名称
- 所在镇街
- 主营产品
- 分类依据
- 数据来源
- 证据片段

**示例：**
输入：

- 某企业官网介绍页面

输出：

- 企业名称：某某科技有限公司
- 所在镇街：桂城街道
- 主营产品：数据分析平台
- 分类依据：官网显示提供大数据分析服务
- 数据来源：官网链接
- 证据片段：提供数据分析与智能决策解决方案

---

### Agent3：分类

**输入：**

- 企业信息
- taxonomy 分类标准

**功能：**

- 根据企业信息判断所属分类
- 给出分类依据和置信度
- 输出结构化分类结果

**输出：**

- 类别
- 分类依据
- 置信度

**示例：**
输入：

- 企业主营业务：数据分析平台、AI建模
- 分类标准：taxonomy.md

输出：

- 类别：数据技术类
- 分类依据：主营业务以数据分析和建模为主
- 置信度：0.92

---

### Agent4：校验

**输入：**

- 企业数据
- 已有数据库数据

**功能：**

- 检查字段缺失
- 检查企业名称重复
- 检查分类冲突
- 检查是否存在低质量或可疑数据

**输出：**

- 是否冲突
- 是否重复
- 是否缺失关键字段
- 是否需要人工复核

**示例：**
输入：

- 企业名称：某某科技有限公司
- 分类：数据服务类
- 来源：1条官网

输出：

- 是否冲突：否
- 是否重复：否
- 是否缺失：否
- 是否需要人工复核：是（来源较少）

---

## 4. 人工复核

**触发条件：**

- 置信度过低
- 分类依据不充分
- 来源数量过少
- 字段冲突或重复
- 企业是否属于数据产业不明确

**人工复核内容：**

- 确认企业是否真实存在
- 确认企业是否属于南海区
- 确认企业分类是否合理
- 确认证据片段是否支持分类结论

**输出：**

- 复核结论：通过 / 修改 / 剔除

## 5. 与当前系统的关系

当前系统已经具备：

- taxonomy 分类体系
- sample_20.csv 样本数据
- FastAPI 后端接口
- React 前端展示页面

Agent 流程将在现有系统基础上用于：

1. 自动扩展更多企业候选数据
2. 半自动生成结构化字段
3. 辅助分类和校验
4. 提高大规模数据集构建效率

## 本项目通过“企业发现 Agent + 信息抽取 Agent + 分类 Agent + 校验 Agent + 人工复核”的协同流程，实现南海区数据产业企业图谱的高质量构建。

## 8. 流程图（文字版）

用户输入关键词与地区
→ Agent1 搜索候选企业
→ Agent2 抽取企业结构化信息
→ Agent3 按 taxonomy 进行分类
→ Agent4 做重复、缺失、冲突校验
→ 人工复核低置信数据
→ 写入数据库
→ 前端展示与筛选查询

---

## 9. Agent 与系统模块对应关系

| Agent           | 当前/未来对应模块       |
| --------------- | ----------------------- |
| Agent1 企业发现 | 搜索脚本、企业候选池    |
| Agent2 信息抽取 | 网页解析、字段抽取脚本  |
| Agent3 分类     | taxonomy + 分类逻辑模块 |
| Agent4 校验     | 去重规则、字段校验逻辑  |
| 人工复核        | 人工检查表、复核记录    |

---

## 10. 设计理由

本项目不直接依赖单一模型一次性完成所有任务，而是拆分为多个 Agent，原因如下：

1. 不同任务的目标不同，拆分后更易控制
2. 企业发现、信息抽取、分类、校验的输入输出不同
3. 多 Agent 协同更适合后续扩展到大规模企业数据
4. 人工复核可以弥补自动化分类的不确定性
5. 更符合比赛中“AI + 数据工程 + 业务规则”的综合能力要求
